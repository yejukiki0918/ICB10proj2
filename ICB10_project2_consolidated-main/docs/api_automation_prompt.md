우리가 사람인 사이트에서 크롤링한 '기획/전략' 직무 채용공고 1,000건 데이터셋(CSV)과 네이버 카페 검색 API(Search API)를 연동하여, "수요-공급 미스매치 분석을 위한 통합 데이터 마트(Data Mart)"를 자동으로 구축하는 파이썬 전처리 및 수집 자동화 파이프라인 코드를 짜줘.

수작업 검색 없이 데이터가 유기적으로 흐르도록 아래 3단계 파이프라인 조건과 예외 처리 로직을 완벽하게 반영한 하나의 파이썬 스크립트를 작성해 줘.

---
### [1단계: 사람인 데이터 기반 핵심 스킬셋 추출 (수요)]
- 파일명 'saramin_1000.csv'를 로드해 줘. (주요 컬럼명: '우대사항')
- 기획/전략 직무의 도메인을 반영하여 분석할 핵심 자격증/스킬 리스트를 사전에 정의해 줘. (예: ['SQLD', 'Figma', '컴퓨터활용능력', 'GA4', 'CFA'])
- 파이썬 내장 기능을 활용해 '우대사항' 텍스트 전체에서 해당 키워드들이 각각 몇 번 언급되었는지 카운트하여 '직무별_기업_수요' 딕셔너리나 데이터프레임으로 마스터화해 줘.

### [2단계: 네이버 카페 검색 API 연동 및 루프 자동화 (공급)]
- 네이버 개발자 센터의 '카페글 검색 API(cafearticle.json)'를 연동하는 함수를 만들어 줘. (client_id와 client_secret 변수 포함)
- 구직자의 순도 높은 데이터만 타겟팅하기 위해, 검색어 쿼리(query) 송신 시 1단계에서 정의한 자격증 키워드 뒤에 " 독취사" 또는 " 스펙업" 같은 대표 취업 카페명을 문자열 결합(f-string)하여 전달하도록 튜닝해 줘.
- 사람이 일일이 검색하는 수작업을 배제하기 위해, 1단계의 핵심 키워드 리스트를 `for문`으로 순회하며 네이버 API에 자동 요청하는 루프(Loop) 코드를 짜줘.
- 네이버 API 트래픽 제한 및 IP 차단(DDoS 오인)을 방지하기 위해, 하나의 키워드 요청이 끝날 때마다 `time.sleep(0.5)`를 적용해 안전장치를 세워줘.

### [3단계: 통합 데이터 마트 정형화 및 CSV 저장]
- API가 반환한 JSON 데이터에서 각 키워드별 '총 게시글 수(total)'를 추출하여 구직자 공급량 지표로 매핑해 줘.
- 동시에 상세 데이터인 게시글 제목(title), 본문 요약(description), 카페명(cafename)을 파싱하여 하나의 Pandas DataFrame으로 결합해 줘.
- 최종적으로 1단계의 사람인 수요 카운트와 2단계의 네이버 카페 공급 총량을 자격증 키워드 기준으로 매칭(Merge)하고, 안티 그래비티 작업 공간에 'automated_cafe_supply.csv' 파일로 자동 내보내기(Export) 되도록 코드를 마무리지어 줘.
---

불필요하게 무거운 NLP 라이브러리는 제외하고, 순수 파이썬 내장 라이브러리(urllib, json, time)와 Pandas만을 활용해 에러 없이 즉시 실행 가능한 완성형 코드로 출력해 줘.

앞서 작성한 파이썬 데이터 파이프라인 코드에, 네이버 데이터랩 검색어 트렌드 API(Search Trend API)를 연동하여 "기획/전략 직무 키워드와 필요 역량 키워드가 조합된 통합검색어 추이"를 자동 수집하고 결합하는 로직을 추가해 줘.

수작업으로 키워드를 조합하여 검색하는 번거로움을 없애기 위해 아래의 자동화 및 데이터 병합 조건을 완벽하게 반영해 줘.

---
### [추가 요구사항: 직무 + 역량 조합어 통합검색 추이 자동 수집]

1. **직무 및 역량 조합어 배열 자동 생성:**
   - 직무 키워드 배열 `['기획직무', '전략기획', '서비스기획']`과 1단계에서 도출된 핵심 역량/자격증 리스트 `['SQLD', 'Figma', '컴활']`를 활용해 줘.
   - 파이썬 루프를 통해 `[직무 키워드 + 역량 키워드]` 조합의 검색어 그룹을 자동으로 생성하도록 코드를 짜줘. (예: "전략기획 SQLD", "서비스기획 Figma" 등)

2. **네이버 데이터랩 검색어 트렌드 API (`/v1/datalab/search`) 연동:**
   - 위에서 조합된 키워드 그룹들을 네이버 데이터랩 API에 파라미터(keywordGroups)로 자동 매핑하여 던지는 요청 함수를 구현해 줘.
   - 조회 기간은 사람인 데이터와 싱크를 맞추기 위해 `2026-01-01`부터 `2026-06-30`까지, 조회 주기는 월간(`month`)으로 설정해 줘.
   - 트래픽 제한 및 IP 차단을 방지하기 위해 검색어 트렌드 API 호출 시에도 `time.sleep(0.5)` 안전장치를 포함해 줘.

3. **데이터 마트 통합 및 다차원 Gap 분석 데이터셋 저장:**
   - 데이터랩 API가 반환한 월간 relative ratio(상대적 검색 빈도) 데이터를 파싱하여 Pandas DataFrame으로 정형화해 줘.
   - 최종적으로 기존 [1단계의 사람인 채용공고 수요 빈도], [2단계의 네이버 취업카페 언급 빈도], 그리고 이번에 추가된 [통합검색 조합어 트렌드 추이 데이터]를 '역량/자격증 키워드' 기준으로 유기적으로 조인(Merge)해 줘.
   - 모든 분석 데이터가 결합된 최종 결과물을 안티 그래비티 작업 공간에 'automated_total_mismatch_mart.csv' 파일로 깔끔하게 내보내기(Export) 하도록 코드를 완성해 줘.
---

기존 파이프라인의 구조적 일관성을 유지하면서, 에러 없이 한 번에 다차원 미스매치 데이터셋을 생성할 수 있는 빌드업 코드로 작성해 줘.
