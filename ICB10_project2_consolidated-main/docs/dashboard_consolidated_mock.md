📊 [통합 완성 기획안] 취업 시장 다차원 EDA 및 직무 적합도 진단 솔루션 (SaaS)
부족한 데이터에 대해서는 mock data 활용하되, 데이터 수급은 자동화가 원칙
mock data를 활용했으면 별도 표기 필수

1. 프로젝트 개요 (Background & Goal)
배경 (Why): 구직자는 채용 시장의 실제 수요와 맞지 않는 정형 스펙(ex. 컴퓨터활용능력)에 시간을 낭비하고, 기업은 변화하는 트렌드에 맞는 인재를 찾지 못하는 정보 비대칭 및 수급 미스매치가 발생함. 특히 기획/전략 직무는 파편화가 심해 매칭에 어려움이 큼.

목표 (Goal): 사람인 공고 데이터 1,000건(수요)과 네이버 취업 카페/통합검색 조합어 API 데이터(공급)를 유기적으로 결합한 '다차원 미스매치 데이터 마트'를 구축함. 이를 기반으로 거시적 시장 탐색(EDA)부터 미스매치 진단, JD(채용공고) 최적화 솔루션까지 한 번에 제공하는 통합 SaaS 대시보드를 구현함.

🧬 2. 데이터 파이프라인 및 전처리 자동화 전략
수요 데이터 (사람인): 기획/전략을 포함한 5대 핵심 직무 공고 1,000건 크롤링 및 2단계 상세 페이지 파싱 완료. 파편화된 직무명을 표준화하고, 복잡한 NLP 대신 순수 파이썬 in 연산 및 Count 기반 매칭으로 로직을 단순화 및 고도화함.

공급 데이터 (네이버 API 자동화): 사람인에서 추출된 마스터 스킬셋을 기반으로 파이썬 루프(Loop)를 가동하여, [독취사/스펙업 등 취업 카페 내 키워드 언급 빈도]와 [직무 키워드 + 역량 키워드 조합어 트렌드 추이]를 실시간 노이즈 필터링하여 통합 데이터 마트로 자동 적재함.

🖥️ 3. Streamlit 마스터 대시보드 아키텍처 (화면 구성안)
🎛️ 사이드바 컨트롤러 (Control Tower)
대시보드 전반을 관통하는 필터링 창구.

직무 선택 셀렉트 박스: ["기획/전략", "인사/노무", "회계/재무", "감사/컴플라이언스", "마케팅", "데이터분석가/AI엔지니어"]

동작: 직무를 스위칭하는 즉시, 탭 1과 탭 2의 내부 데이터프레임과 시각화 맵이 동적으로 리프레시(Reactive)됨.

📊 탭 0. [앞단 배치] 취업 마켓 다차원 EDA 센터
목적: 대시보드 진입 즉시 전체 취업 시장의 통계적 본질과 데이터 신뢰성을 증명하는 간판 화면. 사이드바 필터에 종속되지 않고 전체 시장을 조망함.

① 핵심 요약 지표 (st.columns & st.metric):

분석된 채용공고 수 (1,000+ 건), 추출된 마스터 스킬셋 개수, 필터링된 구직자 행동 데이터 수 시각화.

② 수요-공급 4분면 포지셔닝 맵 (st.plotly_chart 산점도):

X축(기업 공고 수요)과 Y축(구직자 카페/검색 공급량)을 교차하여, 자격증/스킬셋별 포지션 시각화.

인사이트: 2사분면(과공급 위험: 컴활 등 스펙 낭비 구역)과 4사분면(블루오션: Figma 등 채용난 구역)을 시각적으로 명쾌하게 분리 유도.

③ 세부 직무별 스킬 태그 동시 출현(Co-occurrence) 빈도 분석:

동일 공고 내에 세트로 묶여서 올라오는 역량의 군집(Cluster) 시각화 (ex. IT기획 군집: [Figma - 역기획 - 서비스로그 - SQLD]).

④ 시계열 검색량 변동성(Volatility) 분석:

2026년 상반기 조합어 데이터의 월별 추이를 분석하여 공채 시즌 전후 구직자 유입 유효 타이밍 도출.

💡 탭 1. 구직자: 스펙 자가진단 및 스코어링 엔진
목적: 구직자가 본인의 스펙 낭비를 깨닫고 시장의 진짜 수요에 맞출 수 있도록 돕는 B2C 진단 툴.

① 동적 스킬셋 셀렉터 (st.multiselect):

사이드바에서 선택된 직무에 맞는 자격증/스킬 목록이 자동 로드되며 유저가 보유 스펙 선택 가능.

② 매칭 적합도 스코어링 (st.metric):

공식: (유저 보유 스킬 중 공고 언급 개수 / 해당 직무 공고의 평균 요구 스킬 개수) * 100 계산 후 점수 출력.

③ 스펙 선점 우선순위 처방 (st.warning):

현재 유저에게 부족하지만 시장 수요가 가장 높은 상위 키워드 TOP 3를 매칭하여 우선순위 로드맵 제안.

🏢 탭 2. 인사팀: 수급 Gap 분석 및 JD 최적화 도구
목적: 기업 인사담당자에게 허수 지원자를 줄이고 채용 확률을 높이는 액션 플랜을 제공하는 B2B 컨설팅 룸.

① 이중 축 수급 Gap 비교 분석 그래프 (st.bar_chart 등):

선택된 직무의 구직자 관심도(파란색)와 실제 기업 우대 빈도(빨간색)를 대조하여 미스매치 강도를 시각화.

② JD(채용공고) 리모델링 시뮬레이터 (인터랙티브 툴):

인사담당자가 현재 자사 공고의 우대사항 태그들을 multiselect로 입력하면, 시장 데이터 마트와 대조하여 "채용난 위험 역량"인지 "과공급 역량"인지 실시간 판정.

③ 10년 차 수석 기획자의 채용 전략 제언 (st.info / st.success):

직무별 특성에 맞춘 동적 텍스트 처방전 출력.

기획 직무 예시: "인재 부족 영역인 '인수합병/Figma' 인재를 유인하기 위해, 구직자 검색 빈도가 높은 키워드를 공고 상단에 의도적으로 배치(JD Optimization)하고 사내 기획자 양성 로드맵을 선제 공개하십시오."
