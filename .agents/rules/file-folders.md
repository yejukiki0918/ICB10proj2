현재 워크스페이스 안에 폴더를 만들때 다음 규칙으로 구성해 주세요.
프로젝트 폴더는 다음과 같이 생성합니다.

- project_a
- project_b

각 프로젝트 폴더 안에는 다음 하위 폴더를 생성합니다.

- images : 이미지 파일 저장
- docs : 문서 파일 저장
- src : 소스코드 저장
- report : 분석 결과, 보고서, 산출물 저장
- data : 원본 데이터 및 가공 데이터 저장

최종 폴더 구조는 다음과 같아야 합니다.

workspace/
├── .venv/
├── project_a/
│   ├── images/
│   ├── docs/
│   ├── src/
│   ├── report/
│   └── data/
└── project_b/
    ├── images/
    ├── docs/
    ├── src/
    ├── report/
    └── data/

가상환경은 uv를 사용해서 구성합니다.

중요한 규칙은 다음과 같습니다.

1. .venv 폴더는 워크스페이스 최상단에 한 번만 생성합니다.
2. project_a, project_b 같은 개별 프로젝트 하위 폴더 안에는 .venv를 만들지 않습니다.
3. 이미 워크스페이스 최상단에 .venv 폴더가 존재한다면 새로 만들지 말고 기존 가상환경을 그대로 사용합니다.
4. 모든 프로젝트는 워크스페이스 최상단의 공통 .venv를 함께 사용합니다.
5. 파일과 폴더 경로는 가능하면 절대경로가 아니라 워크스페이스 기준 상대경로로 작성합니다.
6. 폴더 생성 전에 현재 위치가 워크스페이스 최상단인지 확인합니다.
7. 불필요한 파일은 생성하지 말고, 요청한 폴더 구조만 생성합니다.

작업 순서는 다음과 같이 진행합니다.

1. 현재 위치가 워크스페이스 루트인지 확인합니다.
2. 워크스페이스 최상단에 .venv가 있는지 확인합니다.
3. .venv가 없다면 다음 명령으로 uv 가상환경을 생성합니다.

uv venv .venv

4. .venv가 이미 있다면 가상환경을 다시 만들지 않습니다.
5. project_a, project_b 폴더를 생성합니다.
6. 각 프로젝트 폴더 안에 images, docs, src, report, data 폴더를 생성합니다.
7. 작업이 끝나면 생성된 폴더 구조를 트리 형태로 출력합니다.

주의사항:

- 각 프로젝트 폴더 안에서 `uv venv` 명령을 실행하지 마세요.
- `project_a/.venv`, `project_b/.venv` 같은 구조를 만들지 마세요.
- `.venv`는 반드시 워크스페이스 최상단에만 있어야 합니다.
- 하위 프로젝트는 공통 가상환경을 사용하는 구조로 유지하세요.
