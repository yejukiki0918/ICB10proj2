# SQLite 사전 집계 기반 대시보드 최적화 완료

대용량 원본 Parquet 파일(850만 건)을 매번 로드하던 방식에서, **SQLite 기반 사전 집계(Pre-aggregation)** 데이터베이스 아키텍처로 최적화 작업을 완료했습니다.

## 주요 변경 사항

### 1. SQLite 데이터베이스 구축
- `build_sqlite.py` 파이썬 스크립트를 통해 `seoul-pops/data/dashboard.db`를 생성했습니다.
- 원본 850만 건 데이터를 아래 두 개의 최적화된 마스터/맵 전용 집계 테이블로 압축 저장했습니다:
  1. `agg_gu`: `기준일ID, 시간대구분, 시군구명, 연령대, 성별` 그룹화 (필터링 차트용 마스터 뷰)
  2. `agg_dong`: `기준일ID, 시간대구분, 시군구명, 행정동명` 그룹화 (지도 동별 표출 전용 뷰)

### 2. Streamlit 대시보드 리팩토링 (`dashboard.py`)
- 기존의 `@st.cache_data` 내부에서 Parquet을 조인하던 로직을 **SQLite 연결 후 `read_sql`로 읽어오는 방식**으로 전면 교체했습니다.
- 850만 건 데이터를 메모리에 들고 있지 않고 약 50만 건, 30만 건의 경량화된 테이블 2개만 로딩하므로 **로딩 속도가 비약적으로 단축**되었습니다.
- 사용자가 필터를 조작하거나 맵에서 구/동 전환, 시간대 슬라이더를 움직일 때 **거의 즉각적으로 렌더링**됩니다.
- Folium 맵 툴팁 연산도 필터링된 경량 데이터(`map_data`)를 활용하여 지연을 최소화했습니다.

현재 띄워두신 로컬 서버에서 페이지를 **새로고침(F5)** 하시면 향상된 속도로 부드럽게 렌더링되는 지도와 대시보드를 바로 경험하실 수 있습니다!
