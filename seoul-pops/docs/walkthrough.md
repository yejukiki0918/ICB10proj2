# 서울 생활인구 데이터 탐색 대시보드 구현 완료

대규모(850만 건) 서울 생활인구 파켓 데이터를 기반으로 한 **기본 EDA Streamlit 대시보드** 구축이 완료되었습니다. `py-streamlit` 스킬의 체크리스트를 준수하여 구현되었습니다.

## 구현 내용 요약

### 1. 효율적인 데이터 파이프라인
- `@st.cache_data`를 활용하여 초기 로딩 시 파켓 데이터(`LOCAL_PEOPLE_DONG_202606_tidy.parquet`) 전체를 메모리에 한 번만 올려 재실행 성능을 최적화했습니다.
- `행정동코드_매핑정보_20241218.xlsx` 파일의 **행자부행정동코드**를 이용하여 조인함으로써, 이해하기 어려운 코드가 아닌 실제 **자치구/행정동명**을 기준으로 분석할 수 있게 구현했습니다.

### 2. UI/UX 및 필터
- 좌측 사이드바에 **기준일**, **자치구**, **연령대**를 다중 선택할 수 있는 필터를 구현하여 사용자가 원하는 데이터를 빠르게 드릴다운(Drill-down)할 수 있도록 했습니다.

### 3. 핵심 KPI 및 통계 분석 (항목 분석 기법)
- **총 인구수, 평균 인구수, 최대 밀집 시간대, 최대 밀집 지역**을 메인 화면 상단 KPI 카드로 배치했습니다.
- 결측치 및 기초 통계량을 제공하는 원본 데이터 요약 탭을 분리 구성하였습니다.

### 4. Plotly 기반 인터랙티브 시각화
세 가지 주요 관점으로 데이터를 시각화했습니다:
1. **시간대별 추이**: `기준일ID` 그룹에 따른 `시간대구분` 라인 차트로 주야간 인구 변화를 파악합니다.
2. **성별 및 연령 구조**: Pie 차트(성비)와 Bar 차트(연령대별)를 병렬 배치하여 인구 구조를 직관적으로 비교할 수 있습니다.
3. **지역별 밀집도 비교**: 선택된 필터 조건 내 인구가 가장 집중되는 **상위 20개 행정동**을 수평 바 차트로 강조했습니다.

## 실행 방법

아래 명령어를 터미널에서 실행하면 브라우저에 대시보드가 열립니다:

```bash
uv run streamlit run seoul-pops/src/dashboard.py
```
